# Логирование
[English](../en/logging.md) | [简体中文](../cn/logging.md) | [繁體中文](../tw/logging.md) | [日本語](../ja/logging.md) | [한국어](../ko/logging.md) | [Español](../es/logging.md) | [Türkçe](../tr/logging.md) | **Русский**

Компонент поддерживает глобальное логирование и декларативное логирование.

## Глобальное логирование

Глобальное логирование по умолчанию отключено (`enable=false`), его нужно активировать вручную. После включения по умолчанию используется стратегия `BASIC` -- выводятся только строки запроса/ответа (с кодом состояния и временем выполнения), накладные расходы минимальны. Конфигурация по умолчанию:

```yaml
retrofit:
  global-log:
    # Включить логирование (по умолчанию false)
    enable: false
    # Глобальный уровень логирования
    log-level: info
    # Глобальная стратегия логирования (по умолчанию BASIC, вывод только строк запроса/ответа)
    log-strategy: basic
    # Агрегировать ли логи запросов
    aggregate: true
    # Имя лога, по умолчанию полное имя класса LoggingInterceptor
    logName: com.github.lianjiatech.retrofit.spring.boot.log.LoggingInterceptor
    # Скрытые чувствительные заголовки запроса в логах
    # По умолчанию маскируются: Authorization, Proxy-Authorization, Cookie, Set-Cookie
    # Примечание: пользовательская конфигурация полностью переопределяет значения по умолчанию, необходимо самостоятельно включить элементы, которые нужно скрыть
    redact-headers:
      - Authorization
      - Proxy-Authorization
      - Cookie
      - Set-Cookie
```

Четыре стратегии логирования:

1. **NONE**: Не выводить логи
2. **BASIC**: Выводить только строки запроса и ответа
3. **HEADERS**: Выводить строки запроса и ответа, а также заголовки запроса/ответа
4. **BODY**: Выводить строки запроса и ответа, заголовки запроса/ответа и тело запроса/ответа (если есть)

## Декларативное логирование

Если логирование нужно только для некоторых запросов, используйте аннотацию `@Logging` на соответствующем интерфейсе или методе.

## Кастомное расширение

Если нужно изменить поведение логирования, можно наследовать `LoggingInterceptor` и зарегистрировать его как Spring Bean.
