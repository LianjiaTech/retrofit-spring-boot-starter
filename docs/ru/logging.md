# Логирование

Компонент поддерживает глобальное логирование и декларативное логирование.

## Глобальное логирование

Глобальное логирование по умолчанию отключено (`enable=false`), его нужно активировать вручную. После включения по умолчанию используется стратегия `BASIC`, которая выводит только строку запроса/ответа (включая код состояния и время выполнения), с незначительными затратами ресурсов. Конфигурация по умолчанию:

```yaml
retrofit:
  global-log:
    # Включить логирование (по умолчанию false)
    enable: false
    # Уровень глобального логирования
    log-level: info
    # Стратегия глобального логирования (по умолчанию BASIC, выводит только строку запроса/ответа)
    log-strategy: basic
    # Агрегировать ли логи запросов
    aggregate: true
    # Имя лога, по умолчанию полное имя класса LoggingInterceptor
    logName: com.github.lianjiatech.retrofit.spring.boot.log.LoggingInterceptor
    # Конфиденциальные заголовки запроса, которые нужно скрыть в логах
    # По умолчанию скрываются: Authorization, Proxy-Authorization, Cookie, Set-Cookie
    # Примечание: пользовательская конфигурация полностью заменяет значения по умолчанию, необходимо самостоятельно включить элементы, которые нужно продолжать скрывать
    redact-headers:
      - Authorization
      - Proxy-Authorization
      - Cookie
      - Set-Cookie
```

Значения четырёх стратегий логирования:

1. **NONE**: не выводить логи
2. **BASIC**: выводить только строку запроса и ответа
3. **HEADERS**: выводить строку запроса и ответа, а также заголовки запроса/ответа
4. **BODY**: выводить строку запроса и ответа, заголовки запроса/ответа, а также тело запроса/ответа (если есть)

## Декларативное логирование

Если логирование нужно только для определённых запросов, можно использовать аннотацию `@Logging` на соответствующих интерфейсах или методах.

## Пользовательское расширение

Если нужно изменить поведение логирования, можно наследовать `LoggingInterceptor` и зарегистрировать его как Spring Bean.
